# DobBot 全自动工作流
# 24小时持续工作，减少人工干预

## 工作流程

### 1. 持续监控（每30分钟）
- 检查 GitHub 推送状态
- 检查 Vercel/Railway 部署状态
- 检查后端 API 健康状态

### 2. 自动构建（代码变更时）
- 检测到代码变更 → 自动构建前端
- 检测到后端变更 → 自动部署 Railway
- 记录构建日志

### 3. 功能开发（按优先级）
- 人声分离（Vocal Split）- 已完成
- 语音转文字（ASR）- 已完成
- 背景移除（Background Removal）- 待完成前端
- 视频下载（Video Download）- 待完成前端
- 后续工具逐个添加

### 4. 日志记录
- 所有操作记录到日志文件
- 只在关键节点通知用户（部署成功/失败）
- 错误自动重试

## 自动化脚本
- 使用 cron/heartbeat 自动执行
- 使用 subagent 并行处理任务
- 使用日志文件记录进度

## 通知策略
- 部署成功 → 通知一次
- 部署失败 → 立即通知 + 自动重试
- 新功能上线 → 通知一次
- 其他情况 → 不通知，记录日志
